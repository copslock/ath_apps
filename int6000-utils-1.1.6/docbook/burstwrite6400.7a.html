NAME
       burstwrite6400 - Burst-Mode Write Only Scope-loop Applet

SYNOPSIS
       int6ktest [memctl.nvm] burstwrite6400.nvm

PURPOSE
       This  scope-loop applet writes all of SDRAM using burst cycles no data is read. It is only
       used to confirm signal integrity of control lines monitored by an oscilloscope.  Look  for
       signal undershoot or overshoot and minimum threshold voltages.

PREREQUISITES
       The  device must be in bootloader mode. SDRAM must be properly configured. See applet mem‚Ä‚Äê
       ctl for one method of configuring SDRAM.

EXAMPLES
       The  following example uses program int6ktest to download and execute applet memctl.nvm to
       configure SDRAM then burstwrite6400 to test memory.

          # int6ktest memctl.nvm burstwrite6400.nvm
          eth2 00:B0:52:00:00:01 Start Applet memctl.nvm (0)
          eth2 00:B0:52:00:00:01 Write Memory (00000040:6076)
          eth2 00:B0:52:00:00:01 Wrote memctl.nvm
          eth2 00:B0:52:00:00:01 Start Applet burstwrite6400.nvm (0)
          eth2 00:B0:52:00:00:01 Write Memory (00001000:692)
          eth2 00:B0:52:00:00:01 Wrote burstwrite6400.nvm
          eth2 00:00:00:00:00:00 Test did not Complete

       Program int6ktest will timeout after 60 seconds if control does not return  to  the  boot‚Äê
       loader so the "Test did not Complete" message is normal in this case.
