NAME
       sdramtest.nvm - SDRAM Diagnostic Applet

SYNOPSIS
       int6ktest [memctl.nvm] sdramtest.nvm [...]

PURPOSE
       This  diagnostic  applet  runs  in  TCM. It performs basic, slow-speed (160ns) SDRAM tests
       lasting about 4 seconds. Control returns to the booloader on successful completion.

FAILURE
       All LEDs will flash slowly on failure. Failure indicates bad SDRAM connections, power sup‚Äê
       ply  out  of specification or major problems with board layout. Look for unconnected pins,
       shorted pins or power problems on the board. Missing or inadequate bypass  capacitors  can
       also  cause failure. If this test fails then most other tests will probably fail. Try run‚Äê
       ning the marchloop applet to help determine the nature of the failure.

PREREQUISITES
       Bootloader software must be running. The SDRAM controller must be properly configured. See
       the memctl applet for one means of configuratin SDRAM.

EXAMPLES
       The  following  example  uses  program  int6ktest to download and execute applet file mem‚Ä‚Äê
       ctl.nvm then applet file sdramtest.nvm. Applet memctl is needed to configure  the  onboard
       SDRAM controller before executing applet nvramtest.

          # int6ktest memctl.nvm sdramtest.nvm
          eth2 00:B0:52:00:00:01 Start Applet memctl.nvm (0)
          eth2 00:B0:52:00:00:01 Write Memory (00000040:6076)
          eth2 00:B0:52:00:00:01 Wrote memctl.nvm
          eth2 00:B0:52:00:00:01 Start Applet sdramtest.nvm (0)
          eth2 00:B0:52:00:00:01 Write Memory (00001000:740)
          eth2 00:B0:52:00:00:01 Wrote sdramtest.nvm
