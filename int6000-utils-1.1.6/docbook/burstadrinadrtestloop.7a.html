NAME
       burstadriadrtestloop - Address-in-Address Scope-loop Applet

SYNOPSIS
       int6ktest [memctl.nvm] burstadriadrtestloop.nvm [...]

PURPOSE
       This  scope-loop  applet writes the address of each memory location in that location using
       burst cycles then reads each location looking for errors. All LEDs blink  amber  at  three
       cycles per second if an error is detected.

       The  applet  executes in Tightly Coupled Memory (TCM) so the test is dependent only on the
       INT6400, SDRAM and the connections between them. SDRAM bus cycle operates at one  half  of
       the INT6400 core execution time.

PREREQUISITES
       The  device must be in bootloader mode. SDRAM must be properly configured. See applet mem‚Ä‚Äê
       ctl.nvm for one method of configuring SDRAM.

EXAMPLES
       The  following example uses program int6ktest to download and execute applet memctl.nvm to
       configure SDRAM then burstadriadrtestloop to test memory.

          # int6ktest memctl.nvm burstadriadrtestloop.nvm
          eth2 00:B0:52:00:00:01 Start Applet memctl.nvm (0)
          eth2 00:B0:52:00:00:01 Write Memory (00000040:6076)
          eth2 00:B0:52:00:00:01 Wrote memctl.nvm
          eth2 00:B0:52:00:00:01 Start Applet burstadriadrtestloop.nvm (0)
          eth2 00:B0:52:00:00:01 Write Memory (00001000:1096)
          eth2 00:B0:52:00:00:01 Wrote burstadriadrtestloop.nvm
          eth2 00:00:00:00:00:00 Test did not Complete

       Program int6ktest will timeout after 60 seconds if control does not return  to  the  boot‚Äê
       loader so the "Test did not Complete" message is normal in this case.
