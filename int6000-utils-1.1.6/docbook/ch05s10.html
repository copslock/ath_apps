Start a Device

Some INT6300™ and INT6400™ device configurations have no NVRAM to store runtime firmware and parameters so the local host must write them into SDRAM and start them. This is also needed when the NVRAM is blank or corrupted because the runtime firmware is used to flash NVRAM. Programs int6kf and int6kboot can be used to configure the SDRAM then download runtime parameters and firmware and start execution on different chipsets.

Example 5.18.  Start Firmware with int6kf 

# int6kf -C abc.cfg -N abc.nvm -P abc.pib

Program int6kf can be used to download runtime firmware and parameters and start execution of the runtime firmware in SDRAM when either the INT6000™ Softloader or theINT6300™ Bootloader is running on the device. The process involves downloading a .cfg file using VS_SET_SDRAM, downloading a .nvm file using VS_WR_MEM, downloading a .pib file using VS_WR_MEM then starting firmware execution using VS_ST_MAC. See the int6kf man page for an explanation of the command line options and arguments used here.


Example 5.19.  Start Firmware with int6kboot 

# int6kboot -N abc.nvm -P abc.pib

Program int6kboot can be used to download runtime firmware and parameters and start execution of the runtime firmware in SDRAM when either the INT6300™ or INT6400™ Bootloader is running on the device. The process involves downloading a .nvm file using VS_WR_MEM, downloading a .pib file using VS_WR_MEM then starting firmware execution using VS_ST_MAC. No .cfg file is needed with this application and and error will occur if one is specified on the command line. See the int6kboot man page for an explanation of the command line options and arguments used here.
