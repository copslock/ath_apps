NAME
       grab - byte grabber

SYNOPSIS
       grab [options] file [file] [...]

       grab [options] [<stdin] [>stdout]

DESCRIPTION
       Extract a range of bytes from a file.

OPTIONS
       -l length
              The  number  of  consective  bytes to read. The length may be expressed in decimal,
              hexadecimal or binary. Decimal is the default. Hexadecimal values  start  with  0x.
              Binary  values  start  with 0b. If offset plus length exceeds the file size then an
              error will occur.

       -o offset
              The file offset of the first byte read. The offset may be expressed in  hexadecimal
              or binary. Hexadecimal is the default. Binary values start with 0b.  If offset plus
              length exceeds the file size then an error will occur.

       -q     Enable quiet mode which has not effect.

       -s     Display the output as a text string instead of a hexadecimal character  stream.  If
              the  selected  data  contains  control  characters then output may be unreadable or
              cause unwanted console behavior.

       -v     Enable verbose mode which has no effect.

       -?,--help
              Print program help summary on stdout.  This  option  takes  precedence  over  other
              options on the command line.

       -!,--version
              Print  program  version  information  on  stdout. This option takes precedence over
              other options on the command line. Usethis option  when  sending  screen  dumps  to
              Intellon  Technical  Support  so  that they know exactly which version of the Linux
              Toolkit you are using.

ARGUMENTS
       file   A filename. No assumptions are made based on filename and no  filename  conventions
              are enforced. More than one filename may appear on the command line. If no filename
              appears then input is read from stdin.

EXAMPLES
       None.
