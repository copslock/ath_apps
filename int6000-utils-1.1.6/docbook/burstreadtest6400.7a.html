NAME
       burstreadtest6400 - Burst-Mode Read Only Scope-loop Applet

SYNOPSIS
       int6ktest [memctl.nvm] burstreadtest6400.nvm

PURPOSE
       This  scope-loop applet reads all of SDRAM using burst cycles looking for a fixed pattern.
       The simplified data is dynamic but runs in very short regular bursts so it creates a  dif‚Äê
       ferent power supply noise from the previous test applets.  Read burst occur every 150ns as
       compared to the longer write then read cycles of the previous tests.

PREREQUISITES
       The device must be in bootloader mode. SDRAM must be properly configured. See applet  mem‚Ä‚Äê
       ctl for one method of configuring SDRAM.

EXAMPLES
       The following example uses program int6ktest to download and execute applet memctl.nvm  to
       configure SDRAM then burstreadtest6400 to test memory.

          # int6ktest memctl.nvm burstreadtest6400.nvm
          eth2 00:B0:52:00:00:01 Start Applet memctl.nvm (0)
          eth2 00:B0:52:00:00:01 Write Memory (00000040:6076)
          eth2 00:B0:52:00:00:01 Wrote memctl.nvm
          eth2 00:B0:52:00:00:01 Start Applet burstreadtest6400.nvm (0)
          eth2 00:B0:52:00:00:01 Write Memory (00001000:1064)
          eth2 00:B0:52:00:00:01 Wrote burstreadtest6400.nvm
          eth2 00:00:00:00:00:00 Test did not Complete

       Program  int6ktest  will  timeout after 60 seconds if control does not return to the boot‚Äê
       loader so the "Test did not Complete" message is normal in this case.
